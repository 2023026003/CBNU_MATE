# CBNU Match - 개발 태스크 목록 (Task List)

PRD를 기반으로 실제 개발을 진행하기 위한 작업 목록입니다. 작업이 진행됨에 따라 `[ ]`를 `[/]` (진행 중), `[x]` (완료)로 변경하며 진척도를 관리합니다.

## Phase 1: MVP (최소 기능 제품) 개발

- `[x]` **1. 프로젝트 초기 세팅**
  - `[x]` 프론트엔드 (React, Next.js 등) 및 백엔드 (Node.js, Spring Boot 등) 스택 선정 및 초기 프로젝트 생성
  - `[x]` 데이터베이스 (MySQL, PostgreSQL 등) 설계 및 ERD 작성
  - `[x]` Github 저장소 생성, 브랜치 전략 수립 및 CI/CD 초기 세팅

- `[x]` **2. 회원가입 및 인증 시스템 (핵심 보안)**
  - `[x]` 충북대 이메일(`@cbnu.ac.kr`, `@chungbuk.ac.kr`) 인증 API 구현 (인증 번호 발송 및 확인)
  - `[x]` JWT 또는 Session 기반 인증(로그인/로그아웃) 기능 구현
  - `[x]` 유저 프로필 생성 (닉네임, 학과, 학번 필수, 프로필 이미지 등 선택)
  - `[x]` 마이페이지 기본 정보 조회 및 수정 기능

- `[ ]` **3. 카테고리별 매칭 게시판 (CRUD)**
  - `[ ]` 매칭 게시글 데이터 모델 및 CRUD API 구현
  - `[ ]` 게시글 작성 폼 UI 구현 (공통 항목: 제목, 내용, 인원, 마감일, 해시태그)
  - `[ ]` 카테고리별 분기 처리: 스포츠(종목/장소), 공모전(직무/링크), 스터디(주제/온오프)
  - `[ ]` 메인 페이지 및 카테고리별 목록 리스트 UI (최신순 등 기본 정렬)
  - `[ ]` 게시글 상세 페이지 UI 및 데이터 연동

- `[ ]` **4. 지원 및 매칭 관리 (기본 단계)**
  - `[ ]` 게시글에 대한 '참여 요청' 기능 (지원하기)
  - `[ ]` 작성자가 지원자 목록(프로필 요약) 확인하는 기능
  - `[ ]` 작성자의 수락/거절 처리 및 상태값 변경
  - `[ ]` 인원 충족 시 '모집 완료' 처리 기능

## Phase 2: 기능 고도화

- `[ ]` **5. 검색 및 복합 필터링**
  - `[ ]` 검색창 및 키워드 기반 검색 기능 (제목, 내용, 해시태그 기반)
  - `[ ]` 카테고리, 모집 상태(모집 중/완료), 정렬(마감임박순, 최신순) 필터 컴포넌트 개발 및 API 적용

- `[ ]` **6. 실시간 안전 채팅 시스템**
  - `[ ]` WebSockets (Socket.io 등) 기반 실시간 1:1 채팅 서버 구축
  - `[ ]` 채팅방 목록 및 채팅 UI 구현
  - `[ ]` '참여 요청' 시 혹은 '수락' 시 작성자와 지원자 간 1:1 채팅방 생성 로직 연결
  - `[ ]` 매칭 성사(모집 완료) 시 해당 팀원들 자동 그룹 채팅방 생성
  - `[ ]` 안 읽은 메시지 표시(배지) 기능

- `[ ]` **7. 신뢰도 시스템 (매너 온도/온도탑)**
  - `[ ]` 모임 종료/마감 후 팀원 간 상호 평가(리뷰/별점) 페이지 구현 (익명)
  - `[ ]` 평가 데이터를 기반으로 매너 온도 점수 환산 로직 개발
  - `[ ]` 프로필에 매너 온도 시각화(온도탑 등) 적용
